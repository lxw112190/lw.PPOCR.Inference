# lw.PPOCR.Inference v1.0.0

`v1.0.0` freezes the public C ABI, the private Runtime ABI, and the model
manifest schema v1.  This release represents a **long-term stability
commitment**.

## Stability promises

### Public C ABI

- `LW_PPOCR_API_VERSION` is permanently `1`.
- All public types, enums, and functions listed in
  [abi-freeze-candidate.md](../abi-freeze-candidate.md) are frozen.
- New fields may only be appended to structs after `reserved_*` padding.
- New enumerators may only be appended to enums.
- No function will be removed.  No function signature will change.
- `LW_PPOCR_ABI_FINGERPRINT` will only change when a backward-compatible
  addition (new field/enum) is made — it serves as a "minimum version" check,
  not a breaking-change signal.

### Private Runtime ABI

- `LW_PPOCR_RUNTIME_API_VERSION` is permanently `1`.
- The `lw_ppocr_runtime_api` function table layout is frozen.
  Extensions use the same append-only rules as the public ABI.

### Model manifest schema

- Schema v1 is permanently frozen.
- Future schema versions (2, 3, …) will coexist — a manifest specifies its
  `schema_version` and a Runtime can support multiple versions.

### Threading and memory contract

- The threading rules documented in `ppocr_api.h` are part of the ABI.
- Memory ownership rules (free results with their creating handle, destroy
  only after all results are freed) are part of the ABI.

## Long-term support

- **Security fixes**: Critical vulnerabilities will be backported for at
  least **2 years** from the v1.0.0 release date.
- **Bug fixes**: Non-security bugs that affect correctness will be addressed
  on a best-effort basis.
- **Platform support**: Windows 10/11 x64 is the primary supported platform.
  New Windows versions will be validated as they reach general availability.
- **Backend SDK updates**: New releases of OpenCV, ONNX Runtime, OpenVINO,
  and TensorRT will be validated and supported through minor version bumps
  without ABI changes.

## What "frozen" does NOT mean

- Backend Runtime implementations may change (performance, bug fixes).
- The packaging layout and ZIP structure may change.
- The .NET binding API may evolve independently (it wraps the C ABI).
- New backends may be added.
- The CLI and Demo UI may change.
- Documentation, examples, and tests will continue to improve.

## Migration from v0.x

v0.8.0 and v0.9.0-rc.1 callers can upgrade to v1.0.0 without source changes.
v0.1.0–v0.3.0 callers should review the ABI freeze candidate list; all types
and functions are identical — only defaults and internal behavior may differ.

## Verification

All four production Runtimes pass integration, stability, golden correctness,
and extended soak tests on the verified hardware configurations listed in the
[compatibility matrix](../compatibility-matrix.md).
