# lw.PPOCR.Inference v1.1.0

v1.1.0 是保持 API v1 向后兼容的功能版本。

## 新功能

- 新增 `lw_ppocr_recognize` 和 `lw_ppocr_recognize_batch`，直接识别已经裁剪好的文字行，不执行检测。
- 新增 `lw_ppocr_recognition`、`lw_ppocr_recognition_result` 和对应释放函数。
- Runtime API 函数表只在 v1 尾部追加识别入口；旧 Runtime 仍可执行完整 OCR，新接口会对旧 Runtime 返回 `LW_PPOCR_STATUS_UNSUPPORTED`。
- OpenCV DNN、DirectML、OpenVINO、TensorRT 四个 Runtime 均支持批量只识别，并继续遵循现有 CLS 开关。
- .NET 增加 `Recognize` / `RecognizeBatch`；Python 示例增加同名接口；C 示例覆盖新 ABI 生命周期。
- WinForms Demo 支持鼠标拖拽框选图片区域，松开后立即进行只识别并显示耗时。
- HTTP 服务新增 `/api/recognize`；`/api/ocr` 返回文字框及阶段耗时。
- 测试网页在原图上绘制识别区域，并显示解码、检测、分类、识别、管线及服务端总耗时。

## 兼容性

- `LW_PPOCR_API_VERSION` 保持为 `1`。
- v1.0.0 的结构体前缀、函数签名和行为不变。
- ABI fingerprint 更新为同一兼容家族内的 `LW_PPOCR_ABI_FINGERPRINT_V1_1`。
- 模型 Schema v1 不变，仍要求 detector 与 recognizer。只识别调用跳过 detector 执行，但当前仍从完整模型包创建引擎。

## 验证

- ABI smoke、stability、forward compatibility 测试覆盖新的识别结果分配与释放。
- 四个真实 Runtime 编译通过。
- OpenCV DNN 真实模型集成测试验证完整 OCR 和两张裁剪图批量只识别。
- .NET 8、.NET Framework 4.7.2 WinForms Demo 与 Python 绑定完成编译/语法检查。
